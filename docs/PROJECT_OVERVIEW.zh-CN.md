# 赛博猫灵（Cyber Cat-Spirit）项目说明

## 一、项目定位与目标人群

赛博猫灵是一款面向“养猫人群与宠物内容创作者”的轻量级交互应用，核心目标是把抽象、主观的“猫咪性格判断”转化为可操作、可分享的数字化体验。

目标用户主要包括：

- 日常养猫用户：希望更系统地理解猫咪行为，减少“看不懂猫”的焦虑。
- 新手铲屎官：缺少经验时，希望快速获得结构化解释与互动引导。
- 宠物社媒创作者：需要有趣、可视化、具人格标签的内容素材（如“猫格画像”“猫娘化身”）。
- 宠物服务从业者（潜在）：可作为沟通辅助工具，用于行为观察和用户教育。

## 二、痛点与用户价值

### 1) 解决了什么痛点

- 痛点一：性格判断过于主观
传统判断常依赖个人经验，缺少统一框架，结论容易漂移。
- 痛点二：行为解释碎片化
用户常见“看到一个行为就下结论”，很难连接到稳定性格特征。
- 痛点三：结果不够可视化、不可传播
只有文字描述时，记忆点弱、分享意愿低。

### 2) 提供的用户价值

- 价值一：从“感性猜测”到“结构化评估”
通过问卷与互动实验的组合，建立可重复的评估流程。
- 价值二：从“单点行为”到“整体猫格”
将行为与人格维度关联，输出更完整的猫格画像。
- 价值三：从“结果阅读”到“内容表达”
提供雷达图与猫娘化身映射，让结果更直观、可展示、可社交传播。
- 价值四：从“结论”到“可解释”
保留阶段评分与完成度信息，提升用户对结果的理解与信任。

## 三、实现方案（简述）

项目采用前后端一体化 Node.js MVP 架构，围绕“评估—映射—解读”三条主链路实现。

### 1) 评估链路：问卷 + 实验 + 融合

- 前端提供 10 题单选问卷与互动实验入口。
- 后端 Personality Engine 计算阶段分数：
  - 问卷建立基础性格底色
  - 实验结果进行动态修正
  - 内部按默认权重融合得到最终猫格向量
- 输出包括基础底色、最终猫格标签、核心特质和完成度等。

### 2) 映射链路：人格到形象

- Avatar Mapping 模块将最终猫格映射为可视化人格化结果。
- 前端以雷达图展示五维特征，并联动展示对应猫娘图片。
- 通过统一的 persona 图片映射配置，支持后续资产扩展与替换。

### 3) 解读链路：行为解释

- 用户可选择行为库动作或输入自定义行为。
- Interpreter 支持两种模式：
  - 连接 LLM 时输出更自然的解释文本
  - 无外部 Key 时回退本地模板，保障核心功能可用性

## 四、核心创新点提炼

### 创新点一：三阶段人格建模，而非单次打分

不是只做“一次性问卷结论”，而是采用“问卷定性 + 实验修正 + 融合输出”的渐进式评估机制，兼顾稳定性与行为动态。

### 创新点二：从数据结果到角色化表达

将传统人格评估结果进一步映射为“猫娘化身 + 语义标签 + 视觉呈现”，增强用户情绪连接和内容传播能力。

### 创新点三：可解释性与趣味性的平衡

既保留评分阶段与完成度等理性信息，也提供风格化叙事与角色语录，兼顾“可信”与“好玩”。

### 创新点四：降级可用的工程策略

在可接入大模型的同时，内置本地回退路径，保证在无外部 API 资源时系统仍可完整运行，提升可部署性与稳定性。

## 五、总结

赛博猫灵的价值不在于“给猫贴标签”，而在于为用户提供一种更有结构、可解释且富有互动感的“理解猫咪”的方式。它把宠物行为认知、人格评估和视觉化表达整合在一个轻量应用中，既适合日常使用，也具备内容传播与产品化扩展潜力。
